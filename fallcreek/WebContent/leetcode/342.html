342. Power of Four

Given an integer (signed 32 bits), write a function to check whether it is a power of 4.

Example: Given num = 16, return true. Given num = 5, return false.

Follow up: Could you solve it without loops/recursion?


Difficulty: Easy

Tag: Bit Manipulation

My Solution

Algorithm1

One simple way of solving this problem is to keep dividing n by 4.

Code1

public class Solution {
    public boolean isPowerOfFour(int n) {
         if(n <= 0)
            return false;
        else
            return (n == 1) || ((n % 4 == 0) && isPowerOfFour(n/4));
    }
}

Algorithm2

If we convert the given number to base 4 and the representation is of the form 100...0, then the number if power of 4.

For example:

10 (base 4) = 4 (base 10)

100 (base 4) = 16 (base 10)

1000 (base 4) = 64 (base 10)

...

Code2

public class Solution {
    public boolean isPowerOfFour(int num) {
	return Integer.toString(num,4).matches("^10*$");
    }
}

Algorithm3

If n is power of 4, then we have

4^x = n
where x >= 0 and x is an integer.

Then we gain

x = log(n) / log(4)

n is a power of four if and only if x is an integer.

Note:This solution is problematic because double operations are not absolutely precise. In problem power of three, this soluion won't be accepted.

Code3

public class Solution {
    public boolean isPowerOfFour(int num) {
       
        if(num <= 0)
            return false;
        
        return (Math.log(num) / Math.log(4)) % 1 == 0;
    }
}

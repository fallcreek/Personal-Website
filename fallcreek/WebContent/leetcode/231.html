My Solution

Algorithm

Actually, I didn't figure out an efficient algorithm when I first tried to solve this problem. My solution is very easy and obvious. We will get the answer easily just by dividing the input number by 2 over and over again.

Code

public class Solution {
    public boolean isPowerOfTwo(int n) {
        if(n == 0)
            return false;
        while(n != 1)
        {
            if(n%2 != 0)
                return false;
            else
                n /= 2;
        }
        return true;
    }
}

Algorithm One

The tags of this problem include "Bit Manipulation", so why don't we use it to solve this problem?

Here are some integers that are power of two and their binary code:

1: 0000 0000 0000 0000 0000 0000 0000 0001

2: 0000 0000 0000 0000 0000 0000 0000 0010

4: 0000 0000 0000 0000 0000 0000 0000 0100

8: 0000 0000 0000 0000 0000 0000 0000 1000

...

1,073,741,824: 0100 0000 0000 0000 0000 0000 0000 0000

It's apparent that their binary code only has one "1".

So we only need to count the number of "1" in an integer's binary code to determine if this integer is a power of two.

Code One

public class Solution {
    public boolean isPowerOfTwo(int n) {
    int count = 0;
    while (n > 0)
    {
        count += (n&0x01);
        n >>= 1;
    }
    return count == 1;

    }
}

Algorithm Two

If n is a power of two, then its binary code only has one "1". In other words, the binary code of (n-1) contains a string of "1"s. For example:

8: 0000 0000 0000 0000 0000 0000 0000 1000

7: 0000 0000 0000 0000 0000 0000 0000 0111

We find that eight and(and operation) seven is zero, and this feature is applicable for every integer that is a power of two.

Thus, we can use and operation to solve this problem easily.

Code Two

public class Solution {
    public boolean isPowerOfTwo(int n) {
        if(n <= 0)
            return false;

        if((n & (n-1)) == 0)
            return true;
        else
            return false;
    }
}
